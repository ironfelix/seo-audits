03 · Технический SEO

Технические проблемы

WordPress + WooCommerce, PHP 7.4 (EOL). Основные технические файлы в порядке, но есть критические on-page пробелы.

P0
Нет H1 на главной странице блокирующее
На главной странице отсутствует тег <h1>. Первый heading — H2: «Optic Space — сеть салонов оптики в Новосибирске, которой доверяют». Это прямая причина, почему «новосибирск оптика» (WS 3685) застрял на позиции 10, а «купить очки новосибирск» (WS 1721) на позиции 16.
Проверено: grep <h1> homepage → 0 совпадений
P0
PHP 7.4.33 — End of Life с декабря 2022 безопасность
Сервер отдаёт x-powered-by: PHP/7.4.33. Версия без security-патчей уже 3,5 года. Риски: уязвимости в WP-плагинах, возможные проблемы с хостингом при принудительном обновлении, потеря производительности vs PHP 8.x.
Источник: HTTP response headers
P1
Cyrillic URL-encoded слаги у большинства товаров высокий
~70% товарных страниц имеют URL вида /product/%d0%be%d0%bf%d1%80%d0%b0%d0%b2%d0%b0-baroncelli-23103-c6/ вместо транслита. Яндекс предпочитает латиницу в slug. Ссылки некликабельны при копировании, шаринг сломан.
Хороший пример: /product/oprava-opticheskaja-1769/ · Плохой: /product/%d0%be%d0%bf%d1%80%d0%b0...
P1
OG-изображение по HTTP, не HTTPS высокий
В мета-теге: og:image: http://optic-space.com/wp-content/uploads/2025/12/Frame-35.png. Mixed content: браузеры и соцсети блокируют HTTP-изображения. Превью при шаринге ВКонтакте/Telegram может не отображаться.
P1
Только 30 страниц в индексе высокий
Видимо, большинство карточек товаров не индексируется. В топ-50 только 1 товарная страница (/product/oprava-opticheskaja-1663 с WS:109). У happylook.ru — 2214 страниц в индексе, у Optic Space — 30. Проверить sitemap и настройки WooCommerce SEO.
Данные: keys.so pagesinindex=30
P2
Schema.org: нужно расширить среднее
Присутствует базовый блок WebSite/Organization. Нет: Product на карточках товаров (цена, наличие, бренд), LocalBusiness с адресом и OpeningHours, BreadcrumbList на категориях. Без Product schema — нет цен в сниппете.
✅
Хорошее: robots.txt, sitemap, canonical, meta robots
robots.txt корректный с Yandex Sitemap-директивой. sitemap_index.xml существует. Canonical на главной правильный. Meta robots: index, follow, max-image-preview:large — всё верно. Это нетипично хорошо для небольшого WP-сайта.

07 · Quick Wins

Быстрые победы (первые 2–4 недели)

Изменения с максимальным ROI и минимальными усилиями.

1. Добавить H1 на главную страницу
Усилие: 30 минут
Impact: «Новосибирск оптика» (WS 3685) с pos:10 → топ-3
→ H1: «Оптика в Новосибирске — очки для зрения и солнцезащитные» или «Optic Space — салон оптики в Новосибирске: очки, линзы, диагностика зрения»
2. Обновить и расширить статью «Очки-хамелеоны» под форму лица
Усилие: 3–4 часа (копирайтер)
Impact: 40K WS/мес кластера, pos:19-24 → топ-10
→ Добавить секции по каждому типу лица с фото, таблицу, FAQ. Текущая статья недогружена контентом.
3. Исправить OG image: HTTP → HTTPS
Усилие: 15 минут (WP настройки)
Impact: Шаринг в соцсетях заработает корректно
→ В WordPress: Настройки → Общие → URL сайта сменить на https:// Или в SEO-плагине вручную поправить og:image.
4. SEO-оптимизация страницы /diagnostika-zreniya/
Усилие: 2–3 часа
Impact: Лид-генератор: диагностика → продажа
→ H1: «Диагностика зрения в Новосибирске», текст 400–600 слов, цена, время, FAQ. Добавить MedicalProcedure schema.
5. Полноценная страница /linzi с SEO-текстом
Усилие: 4–6 часов
Impact: Закрыть пробел 3500+ WS/мес по линзам
→ H1 «Контактные линзы в Новосибирске», категории: однодневные, цветные, для астигматизма. WooCommerce категория с товарами.
6. Обновить PHP 7.4 → 8.2+
Усилие: Хостинг-провайдер, 1–2 часа + тест
Impact: Безопасность, производительность, поддержка плагинов
→ Сначала протестировать на staging. PHP 8.2 даёт прирост скорости 20-30% для WP.
