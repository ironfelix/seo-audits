01 — Executive Summary

Топ-5 критичных проблем

VIS=155 566 — внушительная цифра, но 95% этого трафика генерирует блог «Конверт» по нерелевантным информационным запросам. Email-маркетинговое ядро сайта фактически не оптимизировано, а технические проблемы с hreflang и CWV блокируют рост в коммерческом сегменте.

1

hreflang полностью отсутствует на всём сайте

Сайт имеет URL-структуру с /ru/ префиксом на всех разделах, но ни одна страница не объявляет языковой/региональный таргетинг. Поисковики не знают, что /ru/* — это RU-рынок. Риск: / и /ru/ воспринимаются как конкурирующие URL, PageRank размывается между двумя версиями сайта.

WebFetch crawl.py — ни одного hreflang на главной и всех 5 проверенных страницах, апрель 2026
Критично
2

Глоссарий 468 KB HTML — провальные Core Web Vitals

/ru/glossary/ загружает 400+ терминов, 248 изображений и 1 450 ссылок в одном HTML-файле — в 3–9 раз тяжелее нормы (50–150 KB). Google ранжирует страницы с плохим LCP/CLS ниже конкурентов. Глоссарий критически важен для семантического охвата: 400+ терминов, каждый — потенциальный трафик по информационным запросам.

crawl.py html_size=468 KB на /ru/glossary/ — 248 img, 1 450 ссылок, апрель 2026
Критично
3

95% органического трафика — нерелевантный блог

VIS=155 566, но из 5 000 проанализированных ключей только 81 (1.6%) связан с email-маркетингом. Топ-страницы по трафику: «скачать видео с ютуба» (pos 1), «переводчики через камеру» — не целевая аудитория. Конкуренты notisend (VIS=52, 37 страниц) имеют 53% ключей в email-маркетинге. Выглядит как победа по числам — по факту это провал семантической стратегии.

Высокий
4

Нет FAQPage JSON-LD при наличии FAQ-блоков

/ru/prices/ и /ru/features/email/ имеют полноценные FAQ-секции (5 вопросов каждая), но без FAQPage разметки. Упущены Rich Snippets — FAQ-аккордеоны в SERP увеличивают CTR на 20–30%. Разметка добавляется за 30 минут в Yoast, результат виден через 2–4 недели.

crawl.py: json_ld=[] на /ru/prices/ и /ru/features/email/ — при наличии FAQ-секций в HTML
Высокий
5

OG type = "article" на коммерческих страницах + canonical mismatch

Все коммерческие страницы (главная, цены, возможности, поддержка) имеют og:type = "article" вместо "website". Canonical главной: https://www.unisender.com (без слэша) — фактический URL https://www.unisender.com/. Два разных URL для одной страницы — дублирование сигналов ссылочного веса.

crawl.py: og_type=article, canonical=https://www.unisender.com (без trailing slash)
Высокий

08 — Quick Wins

5 правок за 1 рабочий день

Эти правки не требуют разработки или написания контента. Каждая выполняется за 5–30 минут в Yoast SEO или WordPress шаблоне.

1. FAQPage JSON-LD на /ru/prices/ и /ru/features/email/
FAQ-блоки уже есть на обеих страницах (5 вопросов каждый). Добавить FAQPage разметку в WordPress через Yoast Custom JSON-LD или в head шаблона. Ожидаемо: Rich Snippets FAQ-аккордеоны в SERP через 2–4 недели.
до 30 минвысокий
2. Title главной: 33 → 60 символов
Текущий Title слишком короткий. Замена: «Unisender — сервис email-рассылок и маркетинговой автоматизации» (60 симв.). Одна строка в Yoast SEO → поле Title.
до 5 минсредний
3. Meta description /ru/prices/: 91 → 155 символов
Текущий meta description слишком короткий. Расширить: упомянуть Free/Light/Standard тарифы, конкретные лимиты на отправку, бесплатный период. Один абзац в Yoast SEO.
до 10 минсредний
4. OG type = website на главной, /ru/prices/, /ru/features/, /ru/support/
Исправить og:type с «article» на «website» для всех коммерческих страниц. Правка в Yoast SEO Social → Facebook → Post Object Type, или в functions.php через add_filter('wpseo_og_type', ...).
до 20 минсредний
5. Canonical главной: добавить trailing slash
В Yoast SEO → Search Appearance → General убедиться, что canonical = https://www.unisender.com/ (со слэшем). Устраняет дублирование двух URL главной страницы.
до 5 миннизкий
