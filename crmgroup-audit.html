Топ-3 точки роста по контенту

  1. Починить /email-marketing-pod-klyuch/ — единственная страница под высококонкурентный транзакционный запрос «email маркетинг под ключ» буквально сломана. Нет meta description, двойной H1. После фикса страница получит шанс попасть в топ-10 Яндекса без создания нового контента. Impact: высокий, Effort: минимальный.

  2. Создать страницу /loyalty-program/ — «программа лояльности» (WS: 73 581) — огромный кластер, где в топе SaaS-платформы. Агентство CRM Group с кейсами по настройке программ лояльности может занять позицию «не купите у нас платформу, а настройте с нами». Нет ни одного сильного CRM-агентства в топе по этому запросу.

  3. Статья про структуру email-письма — emailmatrix.ru занял этот кластер (WS: 263 659 для «электронное письмо», WS: 100 786 для «тема письма»). CRM Group — агентство с 400+ клиентами и реальными кейсами design-писем — может создать более экспертную статью с примерами реальных кампаний. Информационный трафик конвертируется через «обсудить вашу рассылку» CTA.

Риски и ограничения

  • DR=16 — сложно конкурировать по высокочастотным коммерческим запросам без прокачки ссылочного профиля. Приоритет — длинный хвост и информационные кластеры, где DR 16-20 достаточен.
  • Закрытые страницы в robots.txt /conditions/ — если это важная страница, ошибка нанесёт урон.
  • 68 статей в блоге — немало контента, но большинство не оптимизированы под конкретные высокочастотные запросы. Нужен аудит существующих статей с переоптимизацией.

Большинство браузеров и поисковых ботов корректно обрабатывают https://host:443/ = https://host/, но это нестандартное поведение. Часть краулеров, особенно устаревших, может обрабатывать их как разные URL, создавая потенциальный дубль. Требует исправления в nginx конфигурации (return 301 https://crmgroup.ru$request_uri; без порта).

Критичные проблемы (топ-5)

  1. Email-маркетинг страница: нет meta description + 2 H1 — /email-marketing-pod-klyuch/ — ключевая коммерческая страница без meta description и с двумя H1-заголовками. Google не может корректно отображать сниппет. При CTR-проверке в выдаче — пустое описание.

  2. HTTP redirect Location: :443 — nginx отдаёт нестандартный Location header https://crmgroup.ru:443/. Потенциальное дублирование канонических URL в некоторых краулерах. Требует одной правки в nginx.

  3. Категории и теги в sitemap и индексе — 39 страниц-архивов (16 категорий + 23 тега) попадают в sitemap без noindex. Это тонкий контент, дублирующий blog listing, тратящий crawl budget и размывающий ссылочный вес.

  4. Массовое отсутствие alt-текста — /special-projects/ (32 из 38 без alt), /web-development/ (10 из 17), /marketing-audit/ (7 из 12). Потеря релевантности для визуального поиска Яндекса + нарушение доступности.

  5. OG type = “article” на коммерческих страницах — /crm-marketing/ имеет og:type = "article". При репосте в мессенджерах карточка отображается как статья, а не как сервис компании. Снижает CTR из социального трафика.

Средние проблемы

  • H1 главной — слоган без ключевых слов
  • H1 /services/ = «Услуги» — без SEO-значения
  • H1 /blog/ — рендер-артефакт «МаркетингНаданных»
  • Title /special-projects/ = 38 символов — слишком короткий
  • Title /telegram-chatbot/, /marketing-audit/ > 70 символов — обрезка в выдаче
  • Blog meta description = 64 символа — слишком короткий
  • Конфликт /conditions в robots vs sitemap
  • testcontent.crmgroup.ru в индексе

Мелочи

  • Двойной Disallow /xmlrpc.php в robots.txt
  • Несогласованные паттерны UTM в robots.txt
  • URL категорий с кириллицей vs латиница — непоследовательно
  • JSON-LD типы не определяются (возможно Organization/WebSite без явного @type)
