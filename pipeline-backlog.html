Thought Leadership формат в /writer — cross-disciplinary lens + coined concepts
Mokka (gomokka.com) использует два трека: SEO/VS-статьи (трафик) + thought leadership (AI citations + бренд). TL-статьи нецитируемы AI-паразитами — у них уникальные концептуальные фреймворки, которые нельзя перефразировать без потери смысла. Сейчас в пайплайне только SEO-статьи. Нет формата, который строит брендовый AI-авторитет.
Как работает формат (паттерн из Mokka):

1. Cross-disciplinary lens — фреймворк из чужой дисциплины
Взять теорию из экономики / поведенческой науки / антропологии / теории игр — и применить к теме клиента. Примеры:
• «Speed-to-hire как ценовой сигнал» → теория сигналов Майкла Спенса (экономика)
• «Cargo cult of employer branding» → антропологический феномен (Фейнман, 1974)
• «Rejection emails as market signals» → principal-agent theory
Для B2B продаж (Factor): «Воронка-заморозка через теорию ликвидности», «КЭВ как снижение информационной асимметрии», «Скрипт-ритуал: когда инструмент теряет смысл».

2. Coined concept — придумать термин для явления
Не описывать — называть. Mokka коинует: «ritual-exploitation economy», «temporal-distributed verification», «the Integrity Funnel», «the 90-second triage».
AI моделям учат новые термины → при поиске этого термина AI цитирует источник. Это прямой Information Gain — нельзя перефразировать.
Тест: если убрать coined concept, статья становится обычной? Тогда концепт слабый.

3. Структура статьи (обязательные 5 элементов):
• Сценарий: конкретная сцена из практики — не «многие компании сталкиваются», а «principal engineer прошёл 4 раунда, оказался deepfake»
• Механизм: почему это системная проблема, а не случайность — применить cross-disciplinary lens
• Coined concept: дать явлению имя с определением
• Quantification: числа с источниками — «detection lag widened from 2.3 days (2023) to 5.7 days (2025)», не «ситуация ухудшается»
• Практический фреймворк: модель с названием (не «5 советов») — «Integrity Funnel: 4 слоя», «Monday-morning audit: 3 шага»

4. Тест на AEO-ценность:
Вставить summary статьи в ChatGPT и спросить: «Кто автор этой идеи?» — если не называет источник, coined concept слабый или информация паразитируема.

Изменения в пайплайне:
• /researcher: добавить шаг «cross-disciplinary angle search» — запрос AI: «какие теории из [economics/psychology/game theory/anthropology] применимы к теме [X]?» → выбрать 2-3 кандидата
• /writer: добавить шаблон TL-статьи (отдельный от SEO-шаблона). Trigger: если статья — «проблема которую никто не считает» / «почему X не работает как принято думать» / «что на самом деле означает Y»
• /quality-gate: новый check — «есть ли coined concept с определением?» — если нет и статья позиционируется как TL, FAIL

Примеры для разных проектов:
• HR/рекрутинг: «The fake-candidate problem nobody is sizing» (Mokka) — $2.6B потерь, coined: «Integrity Funnel»
• B2B продажи: «Воронка-заморозка: почему горячие лиды не закрываются» — через loss aversion + coined concept
• SaaS: «The onboarding tax nobody is measuring» — cost of friction + coined concept
• E-commerce: «Cart abandonment as revealed preference» — behavioral economics

DataForSEO near-miss scoring в PLAN (/researcher)
Mokka PLAN stage: Planner скорит темы из GSC + DataForSEO — search volume, near-miss ranks, competitor gaps — и рефиллит бэклог. Near-miss ranks = статьи на позициях 8–20 которые можно поднять с минимальными усилиями. Это самый высокий ROI контент: работа уже сделана (страница есть), нужен точечный апдейт. Сейчас темы в /researcher выбираются вручную по контент-плану — без алгоритмической приоритизации.
Добавить в /researcher: шаг 0 — topic scoring перед выбором темы.

Scoring formula:
Score = (search_volume × near_miss_mult) + competitor_gap_bonus
• near_miss_mult: поз. 8–10 = 3×, 11–15 = 2×, 16–20 = 1.5×
• competitor_gap_bonus: +50% если конкурент в топ-3, а мы нет по этому ключу

Данные:
• Global: DataForSEO Keywords API + GSC clicks/impressions
• RU: keys.so API (уже подключён) + Яндекс Вебмастер API

Выдача: топ-5 тем с обоснованием: «[тема] — near-miss rank 11, WS 1 200/мес, конкурент в топ-3 → score 89». Выдавать до того как /researcher начинает ресёрч — тема выбрана алгоритмически, не интуитивно.

Orion judge — meta-AI агент для MEASURE (/tracker)
Mokka MEASURE: GSC + 4 AI engines (ChatGPT, Gemini, Perplexity, AI Overview) + GA4, gated by the Orion judge — отдельный AI-агент который решает достаточно ли данных и достаточно ли изменений чтобы передавать сигнал в LEARN. Логика Orion judge (gate перед генерацией action items):

PASS если:
• days_since_publish ≥ 30 (данных накоплено)
• impressions ≥ 100 (статистически значимо)
• |position_delta| > noise_threshold (изменение вне шума, обычно ±3)
• Или: AI citation статус изменился (было — не цитирует, стало — цитирует)

WAIT если:
• Недостаточно данных → записать: «Insufficient data: 14d/38 impressions. Recheck in 16d.»
• Позиции в пределах шума → «Position noise: Δ±2. No action.»

Добавить в /tracker: блок confidence scoring до генерации action items. Если confidence LOW → не генерировать, записать причину в pipeline.md. Если HIGH → генерировать action items как сейчас.

Дополнительно: Orion judge проверяет не только числа но и контекст — если позиция упала но кликов стало больше (SERP изменился) → это не деградация.
