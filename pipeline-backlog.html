SpeakableSpecification в /publisher

В Шаг 2 /publisher добавить к FAQPage-схеме: "@type":"SpeakableSpecification" с cssSelector:[".tl-dr","h2:first-of-type + p"] — лид-абзац и первые предложения первых двух H2. Опционально: HowTo если статья пошаговая.

4
Тире-лимит gate как hard block в /quality-gate
Сейчас тире-лимит (≤8 на статью) есть в правилах /editor и паблишере, но /quality-gate его не блокирует явно. ContentOS имеет языковой gate как обязательный. Добавить автопроверку в quality-gate с точным подсчётом.
Метрика: text.count('—') → если >8 — FAIL. Вывести: Тире: {n}/8 ✗ — вернуть в /editor. Также проверять: нет ли абзаца с 3+ тире. Сейчас эта проверка есть в /editor как «hard gate», но не в /quality-gate как блокирующее условие.

AEO pre-publication scoring в /tracker (до публикации)
ContentOS измеряет AEO-цитируемость до публикации и итерирует (64.2 → 77.1). Сейчас наш /tracker проверяет AI-упоминания только через 30 дней после публикации. Нужна проверка до публикации — хотя бы Perplexity + Yandex AI.

6
AI-detection gate в /quality-gate
ContentOS показывает явный скор AI-детекции (0.28 → likely_human). Это мягкий gate — статья не блокируется, но если скор >0.7 (probably_ai) — возвращается в /editor. Сейчас у нас нет автоматического скора, только правила редактора.
Использовать ZeroGPT API (бесплатный tier: 100 req/день) или GPTZero API. Добавить в /quality-gate метрику 20: AI-detection → если probably_ai (>0.7) — WARN + вернуть в /editor с пометкой конкретных «AI-абзацев». Не блокировать при 0.4–0.7 (ambiguous).
/quality-gate Качество ~3 часа

Citation integrity в /fact-checker
ContentOS автоматически проверяет: каждый тезис с источником семантически следует из этого источника (не просто упомянут, а является логической импликацией). 93.3/100 — 14/15 тезисов верифицированы. Сейчас /fact-checker делает это вручную/по уровням T1-T3.
Добавить в /fact-checker Шаг X: для каждой сноски [источник, тезис] → WebFetch источника → AI-prompt: «Подтверждает ли этот текст тезис: {тезис}? YES/NO/PARTIAL + объяснение». Вывести таблицу: тезис | источник | статус | замечание. Блокировать при 2+ FAIL.

8
Uniqueness check в /quality-gate
ContentOS проверяет уникальность текста (web-плагиат, 0.0 совпадений). Сейчас в пайплайне нет такой проверки. Важно для партнёрских проектов (где /writer может воспроизводить конкурентные тексты) и при batch-публикации 19+ статей подряд.
Использовать text.ru API или Яндекс-текстовый анализ. Добавить в /quality-gate метрику 21: Uniqueness. Порог: уникальность <85% → WARN, <70% → FAIL. Для каждой партии статей запускать перед публикацией.
