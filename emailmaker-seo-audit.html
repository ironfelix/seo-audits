Сводный отчёт

Дата: 2026-06-17 Скоуп: технический SEO + контент/семантика (фокус — рассылки и вёрстка/дизайн писем) + конкуренты и gap-анализ Сервис: EmailMaker — блочный AI-конструктор email-шаблонов (SaaS, B2B, Россия) Профиль keys.so: vis=5 · топ-10: 54 ключа · топ-50: 223 ключа (конкуренты vis 49–177)

Главный вывод

Технических блокировок индексации почти нет — сайт открыт, docs ранжируются. Потолок роста не технический, а контентный: EmailMaker как продукт даёт лучший ответ на запросы «как сверстать письмо / дизайн рассылки / шаблон письма», но не имеет блога и публичной галереи шаблонов — поэтому весь этот трафик (спрос в сотни тысяч показов) забирают блоги конкурентов-ESP (Mindbox, Sendsay, Unisender) и прямые конструкторы (Stripo, Letteros).

Топ-5 критичных проблем

  1. 🔴 robots.txt от чужого движка (OpenCart). Сайт на WordPress, но robots.txt блокирует route=checkout, /catalog, /cart/, /vouchers/ — таких URL нет, а /wp-admin/, /wp-json/ не закрыты. Файл скопирован с другого проекта. → tech-seo-audit.md §1
  2. 🔴 www → 403 вместо 301-редиректа на non-www. Раздвоение хоста. → tech-seo-audit.md §7
  3. 🔴 Нет блога. Вся верхняя воронка (вёрстка, дизайн, подпись письма) недостижима. SERP по этим темам = 100% блоги конкурентов. → content-audit.md §1, §6
  4. 🔴 Шаблоны спрятаны в личном кабинете. Кластер «шаблон писем» — спрос >160 000, мы на 5–18 позициях без индексируемой галереи /templates/. → content-audit.md §3 (кластер 2), gap §5
  5. 🔴 Слабая SEO-упаковка коммерческих лендингов + нет Schema для SaaS. /pricing/ без H2, /cases/ с description «Кейсы», 109/114 картинок на тарифах без alt; нет SoftwareApplication/Organization/FAQPage. → tech-seo-audit.md §5, §9

Что НЕ вошло в аудит (и почему)

  • Core Web Vitals — нет GOOGLE_PSI_API_KEY в .env. Косвенно: главная 314 КБ + 68 картинок + no-cache заголовки → вероятен слабый LCP. Добавить ключ → замерю.
  • ESP-семантика (сервисы/массовая рассылка, доставляемость) — сознательно исключена: EmailMaker не отправляет письма, там не выиграть у Unisender/Sendsay и трафик нецелевой. · gap §6
  • Точные частоты фокус-кластеров «вёрстка»/«дизайн» — keys.so не отдал по точным фразам; спрос подтверждён насыщенным SERP. Уточнить через Wordstat при планировании.
  • Анализ конверсии и аналитики (Метрика/цели) — нет доступа к счётчику клиента.

Следующие шаги

  1. Контент-план на блог — оформить топ точек роста 2–5 в emailmaker/plans/content-plan.md (10–15 статей с H1 и брифами; черновой список — в content-audit.md §5).
  2. ТЗ на галерею /templates/ — структура категорий + индексация (точка роста №1).
  3. Тех-фиксы (robots, www-redirect, schema, alt) — требуют доступа к серверу/WP клиента. Запросить доступы.
  4. Добавить GOOGLE_PSI_API_KEY → дозамерить CWV.

5. Приоритетные направления (топ-3)

  1. Подпись/футер письма (кластер 4.1) — низкая конкуренция, стабильный спрос (~30k суммарно), прямая связка с продуктом (готовый блок подписи). Stripo это уже доит — отъесть долю.
  2. Дизайн + вёрстка письма (кластеры 4.3 + content-audit §4.2–4.3) — высокий информационный спрос, SERP = блоги конкурентов, у EmailMaker лучший продуктовый ответ (шаблоны + AI). Требует блога.
  3. Галерея шаблонов /templates/ (кластер 4.2 + content-audit кластер 2) — коммерческий спрос >160k, уже ранжируемся на 5–20 без выделенной галереи. Самый быстрый коммерческий выигрыш.

6. Что НЕ брать и почему

  • ESP-семантика (сервисы/массовые рассылки, SMTP, доставляемость) — не продукт EmailMaker, проигрыш ESP-гигантам, нецелевой трафик.
  • Служебные пользовательские запросы (confirm email, «прочитано ли сообщение») — нулевая конверсия.
  • Общемаркетинговый блоговый хвост (swot, охват, амбассадор, прямые продажи) — Sendsay набирает им объём, но к конструктору писем это не ведёт.

SERP-разбор по фокус-кластерам «вёрстка» и «дизайн» — в content-audit.md §4.2–4.3 (топ-5 Яндекса: Mindbox, Sendsay, Unisender, SendPulse, Letteros, eSputnik — все блоговые гайды).

6. Выводы

Топ-3 точки роста по контенту: 1. Завести блог. Без него вся верхняя воронка (вёрстка, дизайн, подпись — кластеры 5–7) недостижима. SERP по этим темам = 100% блоги конкурентов-ESP, а EmailMaker как продукт отвечает на них лучше всех (готовые шаблоны + AI). Это и трафик, и естественная конверсия в free trial. 2. Публичная галерея шаблонов /templates/. Кластер «шаблоны писем» — спрос >160k, мы уже на 5–18 позициях без выделенной галереи. Вытащить 250+ шаблонов из личного кабинета в индексируемые категории = самый быстрый коммерческий выигрыш. 3. Консолидировать AI-семантику. Генератор писем (WS 2024, p17) размазан по 3 страницам — собрать в один лендинг.

Риски и ограничения: - Не уходить в ESP-семантику (отправка, доставляемость, базы) — это не продукт EmailMaker, там не выиграть у Unisender/Sendsay. - Блог требует ресурса на регулярный контент — без плана публикаций он не даст эффекта. - Точные частоты фокус-кластеров 5–6 не извлеклись из keys.so — подтверждены косвенно (насыщенный SERP). Уточнить через Wordstat при планировании.

Технический SEO

Дата: 2026-06-17 Статус: анализ (фиксы не применялись — нет доступа к серверу клиента) CMS: WordPress + Yoast SEO + W3 Total Cache, nginx 1.18.0 Источник: crawl.py (5 страниц), curl (robots/sitemap/headers/redirects), WebSearch (индексация)

1. robots.txt

Файл: генерируется (Yoast-блок), отдаётся по /robots.txt.

Проблемы найдены

# Проблема Что видим Делаем
1 🔴 Чужой OpenCart-мусор Disallow: /*route=account/, /*route=checkout/, /*route=product/search, /index.php?route=..., /admin, /catalog, /system, /cart/, /vouchers/, /compare-products/, /add-return/ Это правила для движка OpenCart. Сайт на WordPress — таких URL не существует. robots.txt скопирован с другого сайта. Переписать под WP-структуру
2 🟡 Нет защиты WP-служебки Нет Disallow: /wp-admin/ (кроме admin-ajax), нет /wp-json/ Добавить стандартный WP-блок
3 🟡 Агрессивные параметрические правила Disallow: *page=*, *search=*, /*?sort= и т.п. Часть полезна (фильтры), но *page=* без слеша может задеть пагинацию. Проверить, что не блокирует нужное
4 🟢 Sitemap-директива есть Sitemap: https://emailmaker.ru/sitemap_index.xml ОК, работает
5 🟢 AI-боты не заблокированы User-agent: * без Disallow: / для GPTBot/ClaudeBot/PerplexityBot ОК для LLM-индексации. Можно явно добавить Allow (опционально)

Критичные проблемы (топ-5)

  1. 🔴 robots.txt от чужого движка (OpenCart) — мусорные правила, нет защиты WP-служебки. См. раздел 1.
  2. 🔴 www → 403 вместо 301 на non-www — раздвоение хоста, плохой сигнал. См. раздел 7.
  3. 🔴 Нет блога + слабые коммерческие лендинги (pricing без H2, cases с description «Кейсы», pricing 109/114 картинок без alt) — основной органический потенциал не реализован. Разделы 4–5.
  4. 🔴 Schema для SaaS отсутствует — нет SoftwareApplication/Product/Organization/FAQPage. Теряем rich-результаты и LLM-цитируемость. Раздел 9.
  5. 🟡 Технические дубли и просроченные промо в индексе (test2, main2, blackfriday2024…) — размывают краулинговый бюджет. Разделы 2, 8.

Средние проблемы

  • Массовое отсутствие alt у изображений (главная 55/68, pricing 109/114).
  • llms.txt отсутствует.
  • no-cache заголовки на лендингах (вредит CWV/кэшу).
  • Битые пробелы в заголовках главной (spans без пробела).

Мелочи

  • Description главной длинновата (182).
  • docs без автора/даты.
  • Глубокая вложенность URL в docs.
